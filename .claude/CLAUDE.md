# 多智能体协作全局准则 (针对 Claude)

作为在此项目中工作的智能体，你**必须**严格遵守以下系统级协作规范。本文件旨在解决多 Agent 并发修改导致的状态冲突问题。

## 1. 资源互斥与状态管理 (MCP)

本项目运行了一个名为 `RepoManager` 的独立 C# 状态同步服务（MCP）。
*   **强制锁定**: 在计划写入、编辑或删除工作区内的任何文件时，你必须先向 `RepoManager` 申请对应的路径锁。
*   **阻塞处理**: 若目标文件已被锁定，你必须停止对该文件的操作，并在回复中显式声明“等待资源释放”，或切换至不相关的任务。
*   **及时释放**: 你的 `Commit & Push` 动作完成后，必须立即通知 `RepoManager` 释放资源。

## 2. Git 协作协议 (Skills)

*   项目中包含专门的 Agent 技能（位于 `skills/` 目录下）。
*   在推送代码前，你必须执行标准协作流：确保工作区干净 -> `git pull --rebase` -> 解决潜在冲突（保留主干，注入自身补丁） -> 验证 -> `git push`。
*   **绝对禁止**任何形式的强制推送（Force Push）或直接覆盖他人的最新提交。

## 3. 编码与行为规范

*   根据用户级别的全局规则，所有面向用户的反馈和文档必须使用**中文**。
*   代码单文件严格限制：超过 1000 行时，你必须触发拆分重构逻辑，不能继续在一个庞大文件上堆砌代码。
*   代码必须附带简明扼要的注释（不超过两行）。
*   遇到意图不明的请求，多使用工具进行确认，禁止私自猜测。
